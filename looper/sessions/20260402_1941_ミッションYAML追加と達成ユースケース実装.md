# mission-yaml-and-achieve-usecase: ミッション YAML 追加 + 達成ユースケース + サービス拡張 + アクション統合

## 作成・変更したファイル
- `mission_data/missions.yaml` — `poster-placement` ミッション定義を末尾に追加（difficulty:2, TEXT型, 無制限達成）
- `mission_data/category_links.yaml` — `local-activities` カテゴリに sort_no:160 で追加
- `src/features/poster-placement/use-cases/achieve-poster-placement-mission.ts` — 新規作成。achievement → artifact → XP付与の一連処理
- `src/features/poster-placement/services/poster-placements.ts` — `updatePosterPlacementArtifactId` 関数を追加
- `src/features/poster-placement/actions/poster-placement-actions.ts` — `submitPosterPlacement` にミッション達成フローを統合

## 設計判断
- `achieve-mission.ts` の `processXpGrant` パターンを踏襲しつつ、ポスター掲示専用にシンプル化（max_achievement_countチェック不要、LINK重複チェック不要）
- ミッション達成失敗・紐付け失敗はポスター掲示レコード作成の成功に影響させない設計（ログ記録のみ）
- Use Case層のルールに従い、`adminSupabase` はAction層から引数で受け取る

## 次のタスクへの申し送り
- `missions.yaml` の変更はCI/CDデプロイ時に `npm run mission:sync` で自動反映される。ローカルテスト時は手動実行が必要
- `poster_placements.mission_artifact_id` カラムはMilestone 3のDBマイグレーションで追加済み前提
