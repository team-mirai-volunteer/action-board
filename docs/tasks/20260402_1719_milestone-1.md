#### 1. マイグレーション SQL

以下の内容で作成する。

```sql
-- ポスター掲示テーブルの追加
-- ボランティアがポスター掲示を報告・記録するためのテーブル

-- ==============================================
-- Part 1: poster_placements テーブル
-- ==============================================

CREATE TABLE poster_placements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  lat DECIMAL(10, 8) NOT NULL,
  lng DECIMAL(11, 8) NOT NULL,
  prefecture TEXT,
  city TEXT,
  address TEXT,
  postcode TEXT,
  count INTEGER NOT NULL DEFAULT 1,
  mission_artifact_id UUID REFERENCES mission_artifacts(id) ON DELETE SET NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

COMMENT ON TABLE poster_placements IS 'ポスター掲示の報告レコード';
COMMENT ON COLUMN poster_placements.user_id IS '報告者のユーザーID';
COMMENT ON COLUMN poster_placements.lat IS '掲示場所の緯度';
COMMENT ON COLUMN poster_placements.lng IS '掲示場所の経度';
COMMENT ON COLUMN poster_placements.prefecture IS '都道府県（逆ジオコーディング）';
COMMENT ON COLUMN poster_placements.city IS '市区町村（逆ジオコーディング）';
COMMENT ON COLUMN poster_placements.address IS '住所（逆ジオコーディング）';
COMMENT ON COLUMN poster_placements.postcode IS '郵便番号（逆ジオコーディング）';
COMMENT ON COLUMN poster_placements.count IS '掲示したポスターの枚数';
COMMENT ON COLUMN poster_placements.mission_artifact_id IS 'ミッション連携用の成果物ID';

-- インデックス
CREATE INDEX idx_poster_placements_user_id ON poster_placements(user_id);
CREATE INDEX idx_poster_placements_prefecture_city ON poster_placements(prefecture, city);
CREATE INDEX idx_poster_placements_created_at ON poster_placements(created_at DESC);

-- ==============================================
-- Part 2: RLS ポリシー
-- ==============================================

ALTER TABLE poster_placements ENABLE ROW LEVEL SECURITY;

-- SELECT: 本人のみ閲覧可能
CREATE POLICY "Users can view their own poster placements"
  ON poster_placements
  FOR SELECT
  TO authenticated
  USING (auth.uid() = user_id);

-- INSERT: 認証済みユーザーが自分のレコードを作成
CREATE POLICY "Authenticated users can create poster placements"
  ON poster_placements
  FOR INSERT
  TO authenticated
  WITH CHECK (auth.uid() = user_id);

-- UPDATE: 本人のみ更新可能
CREATE POLICY "Users can update their own poster placements"
  ON poster_placements
  FOR UPDATE
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- DELETE: 本人のみ削除可能
CREATE POLICY "Users can delete their own poster placements"
  ON poster_placements
  FOR DELETE
  TO authenticated
  USING (auth.uid() = user_id);

-- ==============================================
-- Part 3: poster_placement_city_stats 集計ビュー
-- ==============================================

CREATE VIEW poster_placement_city_stats AS
SELECT
  prefecture,
  city,
  SUM(count) AS total_count,
  COUNT(*) AS placement_count
FROM poster_placements
WHERE prefecture IS NOT NULL AND city IS NOT NULL
GROUP BY prefecture, city;

COMMENT ON VIEW poster_placement_city_stats IS '市区町村レベルのポスター掲示集計（個人情報を含まない）';

-- ビューへの SELECT を認証済みユーザーに許可
-- ビューは元テーブルの RLS をバイパスするため、security_invoker を設定
ALTER VIEW poster_placement_city_stats SET (security_invoker = true);

-- ただし service_role（createAdminClient）経由でクエリするため、
-- RLS バイパスで全レコードを集計できる。
-- 認証ユーザーが直接ビューにアクセスする場合は自分のデータしか集計されない点に注意。
```

**重要: `security_invoker` について**

設計仕様書のプライバシー要件では「集計データは認証済みユーザーがSELECT可能」だが、元テーブルの RLS は `user_id = auth.uid()` で本人のみ閲覧可能。ビューで全ユーザー分を集計するには以下のどちらかが必要:

- **方式 A（推奨）**: `security_invoker = true` を設定し、service 層で `createAdminClient()`（service_role）経由でクエリする。service_role は RLS をバイパスするため全レコードを集計できる。
- **方式 B**: `security_invoker` を設定せず（デフォルト = ビュー作成者権限で実行）、ビューが全レコードにアクセスできるようにする。ただしこの場合、クライアントから直接ビューを叩いても全集計が見えてしまうため、意図しないデータ露出のリスクがある。

**→ 方式 A を採用する。** service 層では `createAdminClient()` を使用する。

#### import 先（既存コード）

- `@/lib/types/supabase` → `Database`（自動生成される型）

#### 実装パターン

- **型定義**: `src/features/map-poster/types/poster-types.ts` と同じパターン（`Database["public"]["Tables"][...]` から型を抽出）
- **マイグレーション**: `supabase/migrations/20260131140520_add_youtube_comments.sql` と同じパターン（テーブル作成 → COMMENT → インデックス → RLS 設定）

#### 注意事項

- マイグレーションファイル名のタイムスタンプは `supabase migration new add_poster_placements` コマンドで自動生成すること（手動でタイムスタンプを決めない）
- `supabase.ts` の型再生成後、`pnpm run typecheck` で既存コードに影響がないことを確認する
- `count` カラムは `NOT NULL DEFAULT 1` で、枚数未入力時もデフォルト 1 枚として扱う
- `mission_artifact_id` は `ON DELETE SET NULL` で、成果物が削除されてもポスター掲示レコードは残る
- ビューの `WHERE prefecture IS NOT NULL AND city IS NOT NULL` で、逆ジオコーディング失敗レコードは集計から除外する

#### 1. CRUD サービス

```typescript
// src/features/poster-placement/services/poster-placements.ts
import "server-only";

import { createAdminClient } from "@/lib/supabase/adminClient";
import type {
  PosterPlacement,
  PosterPlacementInsert,
} from "../types/poster-placement-types";

/**
 * ポスター掲示を作成する
 *
 * @param placement - 作成するポスター掲示データ（user_id, lat, lng, count は必須。
 *   prefecture, city, address, postcode は呼び出し元で逆ジオコーディング済みの値を渡す）
 * @returns 作成されたレコード
 */
export async function createPosterPlacement(
  placement: PosterPlacementInsert,
): Promise<PosterPlacement> {
  const supabase = await createAdminClient();

  const { data, error } = await supabase
    .from("poster_placements")
    .insert(placement)
    .select()
    .single();

  if (error) {
    console.error("Error creating poster placement:", error);
    throw error;
  }

  return data;
}

/**
 * 指定ユーザーのポスター掲示一覧を取得する（新しい順）
 *
 * @param userId - ユーザーID
 * @returns ポスター掲示の配列
 */
export async function getPosterPlacementsByUserId(
  userId: string,
): Promise<PosterPlacement[]> {
  const supabase = await createAdminClient();

  const { data, error } = await supabase
    .from("poster_placements")
    .select("*")
    .eq("user_id", userId)
    .order("created_at", { ascending: false });

  if (error) {
    console.error("Error fetching poster placements:", error);
    throw error;
  }

  return data;
}

/**
 * ポスター掲示を ID で取得する
 *
 * @param id - ポスター掲示ID
 * @returns ポスター掲示レコード、見つからない場合は null
 */
export async function getPosterPlacementById(
  id: string,
): Promise<PosterPlacement | null> {
  const supabase = await createAdminClient();

  const { data, error } = await supabase
    .from("poster_placements")
    .select("*")
    .eq("id", id)
    .single();

  if (error) {
    if (error.code === "PGRST116") {
      // Not found
      return null;
    }
    console.error("Error fetching poster placement:", error);
    throw error;
  }

  return data;
}

/**
 * ポスター掲示を削除する
 *
 * @param id - 削除するポスター掲示のID
 */
export async function deletePosterPlacement(id: string): Promise<void> {
  const supabase = await createAdminClient();

  const { error } = await supabase
    .from("poster_placements")
    .delete()
    .eq("id", id);

  if (error) {
    console.error("Error deleting poster placement:", error);
    throw error;
  }
}

/**
 * 指定ユーザーのポスター掲示合計枚数を取得する
 *
 * @param userId - ユーザーID
 * @returns 合計掲示枚数
 */
export async function getUserPosterPlacementCount(
  userId: string,
): Promise<number> {
  const supabase = await createAdminClient();

  const { data, error } = await supabase
    .from("poster_placements")
    .select("count")
    .eq("user_id", userId);

  if (error) {
    console.error("Error fetching user poster placement count:", error);
    throw error;
  }

  return data.reduce((sum, row) => sum + (row.count ?? 0), 0);
}
```

#### 2. 集計クエリサービス

```typescript
// src/features/poster-placement/services/poster-placement-stats.ts
import "server-only";

import { createAdminClient } from "@/lib/supabase/adminClient";
import type { PosterPlacementCityStats } from "../types/poster-placement-types";

/**
 * 市区町村レベルのポスター掲示集計を取得する
 * createAdminClient（service_role）を使用して RLS をバイパスし、全ユーザー分を集計する
 *
 * @returns 市区町村ごとの集計データ配列
 */
export async function getCityStats(): Promise<PosterPlacementCityStats[]> {
  const supabase = await createAdminClient();

  const { data, error } = await supabase
    .from("poster_placement_city_stats")
    .select("*");

  if (error) {
    console.error("Error fetching city stats:", error);
    throw error;
  }

  return data;
}

/**
 * 特定の都道府県の市区町村レベル集計を取得する
 *
 * @param prefecture - 都道府県名
 * @returns 該当都道府県の市区町村ごとの集計データ配列
 */
export async function getCityStatsByPrefecture(
  prefecture: string,
): Promise<PosterPlacementCityStats[]> {
  const supabase = await createAdminClient();

  const { data, error } = await supabase
    .from("poster_placement_city_stats")
    .select("*")
    .eq("prefecture", prefecture);

  if (error) {
    console.error("Error fetching city stats by prefecture:", error);
    throw error;
  }

  return data;
}

/**
 * ポスター掲示の全国合計を取得する
 *
 * @returns { totalCount: 合計枚数, placementCount: 報告件数 }
 */
export async function getTotalStats(): Promise<{
  totalCount: number;
  placementCount: number;
}> {
  const supabase = await createAdminClient();

  const { data, error } = await supabase
    .from("poster_placement_city_stats")
    .select("total_count, placement_count");

  if (error) {
    console.error("Error fetching total stats:", error);
    throw error;
  }

  return {
    totalCount: data.reduce(
      (sum, row) => sum + (Number(row.total_count) || 0),
      0,
    ),
    placementCount: data.reduce(
      (sum, row) => sum + (Number(row.placement_count) || 0),
      0,
    ),
  };
}
```

#### 実装パターン

- **CRUD サービス**: `src/features/map-posting/services/posting-shapes.ts` と同じパターン（`import "server-only"` + `createAdminClient()` + Supabase クエリビルダー）
- **集計サービス**: `src/features/map-poster/services/poster-boards-stats.ts` と同じパターン（ビューからの SELECT + 集計計算）
- **エラーハンドリング**: `console.error` + `throw error` のパターン（既存サービスと統一）

#### 注意事項

- **必ず `import "server-only"` をファイル先頭に記述する**（アーキテクチャガイドライン必須）
- `createAdminClient()` は `await` が必要（async 関数）
- `getPosterPlacementById` で見つからない場合は PostgREST エラーコード `PGRST116` をチェックして `null` を返す
- 集計ビューへのクエリは `createAdminClient()`（service_role）を使う。`createClient()` だと RLS により自分のデータしか集計されない
- `count` カラムは Supabase の型では `number` になるが、ビューの `total_count` / `placement_count` は `SUM` / `COUNT` の結果なので `number | null` になる可能性がある。`Number()` で変換する
