# ポスター掲示アクションボード 設計

## 目的

**ボランティア**が**ポスター掲示の活動を簡単に報告・記録し、掲示状況を地図上で可視化**できるようにするため。

- ボランティアが地図上でピンを打つだけでポスター掲示を報告できる
- 管理側が市区町村単位でポスター掲示状況を集計・把握できる
- 個人情報（貼った人の名前）は公開しない設計とする

## 既存機能との関係

| 機能 | URL | 用途 | 今回との関係 |
|------|-----|------|-------------|
| ポスティングマップ | `/map/posting` | チラシ配布エリアをポリゴンで管理 | 参考（Leaflet+Geoman利用） |
| ポスター掲示板マップ | `/map/poster` | 公営掲示板のステータス管理 | 別機能（掲示板の場所管理） |
| **ポスター掲示マップ（新規）** | `/map/poster-placement` | 個人ポスター掲示の報告・集計 | **今回の設計対象** |

今回は既存2機能とは**独立した新機能**として作成する。

## 機能要件

### ユーザーフロー

1. 右上ドロップダウンメニューから「ポスター掲示マップ」を選択
2. 地図画面が表示される（ログイン必須）
3. 地図上でタップ/クリックしてピンを配置
4. 登録フォームに枚数を入力して送信
5. 掲示情報が保存され、集計に反映される

### 管理項目

| 項目 | 説明 | 入力方式 |
|------|------|---------|
| 掲示場所（緯度・経度） | 地図上でピンを打って指定 | 地図タップ |
| 住所（都道府県・市区町村・番地） | 緯度・経度から逆ジオコーディングで自動取得 | 自動 |
| 枚数 | 掲示したポスターの枚数 | 手入力（数値） |
| 報告者 | ログインユーザーのプロフィール名を自動取得 | 自動（user_id） |

### プライバシー要件

- **個人情報は非公開**: 掲示を報告した人の名前・IDは他ユーザーに一切表示しない
- **公開する集計データ**: 市区町村レベルの掲示枚数集計のみ
- **地図上のピン**: 個別のピン位置は表示しない。市区町村マーカー（集計値付き）のみ表示
- **自分の掲示履歴**: 本人のみ閲覧可能（マイページ等から）

### 集計画面の表示

- 地図上に市区町村マーカーを表示し、集計枚数をラベルとして表示
- マーカーのサイズまたは色で枚数の多寡を視覚的に区別

## データベース設計

### 新規テーブル: `poster_placements`

| カラム | 型 | 説明 |
|--------|-----|------|
| `id` | `UUID` (PK) | 主キー |
| `user_id` | `UUID` (FK → auth.users) | 報告者のユーザーID |
| `lat` | `DECIMAL(10, 8)` | 緯度 |
| `lng` | `DECIMAL(11, 8)` | 経度 |
| `prefecture` | `TEXT` | 都道府県（逆ジオコーディング） |
| `city` | `TEXT` | 市区町村（逆ジオコーディング） |
| `address` | `TEXT` | 住所（逆ジオコーディング） |
| `postcode` | `TEXT` | 郵便番号（逆ジオコーディング） |
| `count` | `INTEGER` | 掲示枚数（デフォルト: 1） |
| `mission_artifact_id` | `UUID` (FK → mission_artifacts, nullable) | ミッション連携用 |
| `created_at` | `TIMESTAMPTZ` | 作成日時 |
| `updated_at` | `TIMESTAMPTZ` | 更新日時 |

### RLSポリシー

| 操作 | ポリシー |
|------|---------|
| SELECT（個別レコード） | 本人のみ（`user_id = auth.uid()`） |
| INSERT | 認証済みユーザー（`user_id = auth.uid()`） |
| UPDATE | 本人のみ |
| DELETE | 本人のみ |

### 集計用ビュー: `poster_placement_city_stats`

市区町村レベルの集計を提供するビュー（またはDB関数）。

| カラム | 説明 |
|--------|------|
| `prefecture` | 都道府県 |
| `city` | 市区町村 |
| `total_count` | 合計掲示枚数 |
| `placement_count` | 報告件数 |

- RLS: 認証済みユーザーがSELECT可能
- 個人を特定できる情報（user_id等）は含めない

## ミッション連携

### 新規ミッション

`mission_data/missions.yaml` に以下を追加:

| 項目 | 値 |
|------|-----|
| slug | `poster-placement`（仮） |
| title | 例:「ポスターを掲示しよう」 |
| difficulty | 要検討 |
| required_artifact_type | 新規タイプ `POSTER_PLACEMENT` または既存 `TEXT` を流用 |
| is_hidden | `false`（公開） |
| is_featured | 要検討 |

### 達成フロー

1. ユーザーがポスター掲示を地図上で報告
2. `poster_placements` にレコード挿入
3. 同時に `achievements` + `mission_artifacts` にレコード作成
4. `poster_placements.mission_artifact_id` で紐付け
5. XP付与・レベルアップ等の既存ミッションシステムに乗る

## ページ構成

### `/map/poster-placement`（新規ページ）

| 要素 | 説明 |
|------|------|
| 地図（Leaflet） | 市区町村集計マーカーを表示。タップでピン配置 |
| 登録フォーム | ピン配置後に枚数を入力して送信するダイアログ/パネル |
| 集計表示 | 地図上のマーカーに市区町村の合計枚数を表示 |

- 認証必須ページ（未ログインはリダイレクト）

## 再利用する既存リソース

| リソース | 場所 | 用途 |
|---------|------|------|
| Leaflet | 既存依存関係 | 地図表示 |
| 逆ジオコーディング | `src/features/map-posting/services/reverse-geocoding.ts` | 住所自動取得 |
| ミッションシステム | `src/features/mission-detail/` | 達成記録・XP付与 |
| DropdownMenu | `src/components/ui/dropdown-menu.tsx` | ナビゲーション |

※ 逆ジオコーディング関数は `poster-placement` から利用するため、必要に応じて共通化（`src/lib/utils/` への移動等）を検討する。

## 必要な環境変数

新規の外部サービス連携なし。追加の環境変数は不要。
