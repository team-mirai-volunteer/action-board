# Milestone 2: ポスター掲示マップページ（地図表示・ピン配置・登録フォーム・ナビゲーション）

## 概要

`/map/poster-placement` ページを新規作成し、Leaflet 地図上でピンを配置して枚数を入力・送信するとポスター掲示が DB に保存される機能を実装する。逆ジオコーディングで住所を自動取得し、右上ドロップダウンからの導線と未ログイン時のリダイレクトも含む。

## タスク一覧

| ステップ | タスクID | Wave | 概要 |
|---------|---------|------|------|
| A | server-action-and-nav | 1 | Server Action（登録・削除）+ ナビゲーションリンク追加 |
| B | map-page-and-components | 2 | ページ・地図コンポーネント・登録フォーム・地図操作フック |

- `reverseGeocode` は `src/features/map-posting/services/reverse-geocoding.ts` にある。このファイルには `import "server-only"` がないため、Server Action から直接 import して呼び出せる。ただし Nominatim API は 1 秒 1 リクエスト制限があるため、連打防止はクライアント側で行う
- `submitPosterPlacement` は逆ジオコーディング結果が null でも DB 保存する（prefecture/city/address/postcode が null のレコードとなる。集計ビューでは WHERE 句で除外される）

#### 実装パターン

- **ページ（page.tsx）**: `src/app/map/posting/page.tsx` と同じパターン（`getUser()` + `redirect("/sign-in")`）。ただしイベント切り替えは不要で、シンプルに `PosterPlacementPageClient` を返すだけ
- **クライアント統合**: `src/features/map-posting/components/posting-page.tsx` を参考にするが、Geoman は不要。地図の dynamic import + useState によるフォーム表示切り替えのみ
- **地図コンポーネント**: `src/features/map-posting/components/geoman-map.tsx` をベースに簡略化。Geoman 関連を除去し、クリックイベントでピン配置するだけ
- **フォーム**: このプロジェクト独自。シンプルなカード型オーバーレイ
- **フック**: `src/features/map-posting/hooks/use-current-location.ts` を参考に、状態管理 + Server Action 呼び出しをまとめる

#### 注意事項

- **Leaflet の dynamic import が必須**: Leaflet は `window` オブジェクトに依存するため、`import("leaflet")` を useEffect 内で行う。トップレベル import は型のみ（`import type { Map as LeafletMap } from "leaflet"`）
- **タイルレイヤー**: `geoman-map.tsx` にはタイルレイヤーの追加がない（Geoman が自動追加する）。poster-placement-map.tsx では `L.tileLayer(...)` を明示的に追加する必要がある。`poster-map.tsx`（14行目付近）を参考にする
- **Leaflet CSS**: `import "leaflet/dist/leaflet.css"` を poster-placement-map.tsx のトップレベルで行う（`poster-map.tsx` と同じパターン）
- **`useCurrentLocation` の依存**: このフックは `(window as LeafletWindow).L` を参照する。地図初期化時に `(window as any).L = L;` を設定する必要がある（`posting-page.tsx` 内で設定されている既存パターンを参照）
- **連打防止**: `isSubmitting` state で送信ボタンを disable にする。Nominatim API の 1 秒制限はサーバーサイドで自然に守られる（1 回の送信 = 1 回のリクエスト）
- **フォームのオーバーレイ**: z-index を地図（z-0）より高く設定。地図のローディング/エラー表示（z-10）より高い z-20 程度
- **`CONTENT_HEIGHT`**: `src/lib/constants/layout.ts` で定義済み。`calc(100vh - 96px)`。地図の高さに使う
