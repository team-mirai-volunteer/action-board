# Milestone 3: 市区町村レベル集計マーカー表示（プライバシー保護）

## 概要

`/map/poster-placement` の地図上に市区町村レベルの集計マーカー（合計枚数ラベル付き）を表示する。個別のピン位置や報告者名は他ユーザーに表示せず、市区町村単位の集計情報のみを可視化する。

## タスク一覧

| ステップ | タスクID | Wave | 概要 |
|---------|---------|------|------|
| A | city-stats-view-and-loader | 1 | 集計ビュー更新（座標追加）+ 型再生成 + loader 作成 |
| B | city-stats-markers-ui | 2 | ページ・クライアント・地図コンポーネント・フック・マーカーユーティリティの実装 |

## タスク詳細

### ステップ A: city-stats-view-and-loader（Wave 1）

集計ビュー `poster_placement_city_stats` にマーカー配置用の平均座標カラムを追加し、Supabase 型を再生成した上で、クライアントから呼び出せる loader を作成する。

#### 作成するファイル

1. `supabase/migrations/YYYYMMDDHHMMSS_update_poster_placement_city_stats_view.sql`（タイムスタンプは `supabase migration new update_poster_placement_city_stats_view` で自動生成）
2. `src/lib/types/supabase.ts`（`pnpm supabase gen types typescript --local > src/lib/types/supabase.ts` で再生成）
3. `src/features/poster-placement/loaders/poster-placement-loaders.ts`

#### 1. マイグレーション SQL

```sql
-- poster_placement_city_stats ビューに平均座標カラムを追加
-- 市区町村集計マーカーの地図上の配置位置に使用する

DROP VIEW IF EXISTS poster_placement_city_stats;

CREATE VIEW poster_placement_city_stats AS
SELECT
  prefecture,
  city,
  SUM(count) AS total_count,
  COUNT(*) AS placement_count,
  AVG(lat)::DECIMAL(10, 8) AS avg_lat,
  AVG(lng)::DECIMAL(11, 8) AS avg_lng
FROM poster_placements
WHERE prefecture IS NOT NULL AND city IS NOT NULL
GROUP BY prefecture, city;

COMMENT ON VIEW poster_placement_city_stats IS '市区町村レベルのポスター掲示集計（個人情報を含まない）';
ALTER VIEW poster_placement_city_stats SET (security_invoker = true);
```

#### マイグレーション適用 + 型再生成

```bash
# ローカル Supabase にマイグレーション適用
pnpm supabase db reset
# または
pnpm supabase migration up

# 型再生成
pnpm supabase gen types typescript --local > src/lib/types/supabase.ts
```

型再生成後、`PosterPlacementCityStats`（`src/features/poster-placement/types/poster-placement-types.ts`）は `Database["public"]["Views"]["poster_placement_city_stats"]["Row"]` を参照しているため、自動的に `avg_lat` と `avg_lng` が追加される。型定義ファイルの手動変更は不要。

#### 実装パターン

- **マイグレーション**: Milestone 1 の `supabase/migrations/..._add_poster_placements.sql` と同じパターン。今回は `DROP VIEW IF EXISTS` → `CREATE VIEW` で再作成

#### 注意事項

- マイグレーションファイル名のタイムスタンプは `supabase migration new update_poster_placement_city_stats_view` コマンドで自動生成すること
- `DROP VIEW IF EXISTS` で既存ビューを削除してから再作成する。ビューに依存するマテリアライズドビューやトリガーはないため安全
- `AVG(lat)::DECIMAL(10, 8)` のキャストで元テーブルと同じ精度を維持する
- `security_invoker = true` を忘れずに再設定する（DROP で消えるため）
- 型再生成後、`pnpm run typecheck` で既存コードに影響がないことを確認する
- loader ファイル先頭は `"use server"` とする（`import "server-only"` ではない。クライアントコンポーネントから Server Action として呼び出すため）

```typescript
// src/features/poster-placement/hooks/use-poster-placement-map.ts
"use client";

import { useCallback, useState } from "react";
import { toast } from "sonner";
import { submitPosterPlacement } from "../actions/poster-placement-actions";

// オプション型（ファイル内定義）
type UsePosterPlacementMapOptions = {
  onSubmitSuccess?: () => void | Promise<void>;
};

type UsePosterPlacementMapReturn = {
  selectedPosition: { lat: number; lng: number } | null;
  isFormOpen: boolean;
  isSubmitting: boolean;
  handlePinPlaced: (lat: number, lng: number) => void;
  handleSubmit: (count: number) => Promise<void>;
  handleCancel: () => void;
};

export function usePosterPlacementMap(
  options?: UsePosterPlacementMapOptions,
): UsePosterPlacementMapReturn {
  const [selectedPosition, setSelectedPosition] = useState<{
    lat: number;
    lng: number;
  } | null>(null);
  const [isFormOpen, setIsFormOpen] = useState(false);
  const [isSubmitting, setIsSubmitting] = useState(false);

  const handlePinPlaced = useCallback((lat: number, lng: number) => {
    setSelectedPosition({ lat, lng });
    setIsFormOpen(true);
  }, []);

  const handleSubmit = useCallback(
    async (count: number) => {
      if (!selectedPosition) return;
      setIsSubmitting(true);
      try {
        const result = await submitPosterPlacement({
          lat: selectedPosition.lat,
          lng: selectedPosition.lng,
          count,
        });
        if (result.success) {
          toast.success("ポスター掲示を登録しました");
          setSelectedPosition(null);
          setIsFormOpen(false);
          // 登録成功後のコールバック（集計データ再フェッチ等）
          await options?.onSubmitSuccess?.();
        } else {
          toast.error(result.error);
        }
      } catch {
        toast.error("登録に失敗しました");
      } finally {
        setIsSubmitting(false);
      }
    },
    [selectedPosition, options],
  );

  const handleCancel = useCallback(() => {
    setSelectedPosition(null);
    setIsFormOpen(false);
  }, []);

  return {
    selectedPosition,
    isFormOpen,
    isSubmitting,
    handlePinPlaced,
    handleSubmit,
    handleCancel,
  };
}
```

**変更点**:
- `UsePosterPlacementMapOptions` 型を追加（`onSubmitSuccess` コールバック）
- `usePosterPlacementMap` の引数に `options?: UsePosterPlacementMapOptions` を追加
- `handleSubmit` の成功後に `await options?.onSubmitSuccess?.()` を呼び出し
- `handleSubmit` の依存配列に `options` を追加

変更なし（既存 import のまま）

```typescript
// src/features/poster-placement/utils/city-stats-marker.ts

/**
 * 市区町村集計マーカーのサイズを合計枚数に応じて決定する
 * 枚数が多いほど大きなマーカーで視覚的に区別する
 */
function getMarkerSize(totalCount: number): number {
  if (totalCount < 5) return 36;
  if (totalCount < 20) return 44;
  if (totalCount < 100) return 52;
  return 60;
}

/**
 * 市区町村集計マーカーの背景色を合計枚数に応じて決定する
 * 枚数が多いほど暖色に変化
 */
function getMarkerColor(totalCount: number): string {
  if (totalCount < 5) return "#60A5FA"; // blue-400（少）
  if (totalCount < 20) return "#34D399"; // emerald-400（中）
  if (totalCount < 100) return "#FBBF24"; // amber-400（多）
  return "#F87171"; // red-400（非常に多）
}

/**
 * 市区町村集計マーカーの Leaflet divIcon を作成する
 *
 * @param L - Leaflet ライブラリ（dynamic import で取得したもの）
 * @param totalCount - 合計掲示枚数
 * @param cityName - 市区町村名（現在未使用だが将来のツールチップ等に備えて受け取る）
 * @returns Leaflet DivIcon インスタンス
 *
 * 既存パターンの参考:
 * - src/features/map-posting/utils/cluster-icon.ts の createClusterIcon
 * - src/features/map-poster/utils/marker-icons.ts の createMarkerIconHtml
 */
// biome-ignore lint/suspicious/noExplicitAny: Leaflet type from dynamic import
export function createCityStatsMarkerIcon(
  L: any,
  totalCount: number,
  _cityName: string,
) {
  const size = getMarkerSize(totalCount);
  const color = getMarkerColor(totalCount);
  const fontSize =
    size <= 36 ? "11px" : size <= 44 ? "12px" : size <= 52 ? "13px" : "14px";

  return L.divIcon({
    html: `
      <div style="
        background-color: ${color};
        width: ${size}px;
        height: ${size}px;
        border-radius: 50%;
        border: 3px solid white;
        box-shadow: 0 2px 6px rgba(0,0,0,0.3);
        display: flex;
        align-items: center;
        justify-content: center;
        color: white;
        font-weight: bold;
        font-size: ${fontSize};
        text-shadow: 1px 1px 2px rgba(0,0,0,0.5);
      ">${totalCount}</div>
    `,
    className: "city-stats-marker",
    iconSize: [size, size],
    iconAnchor: [size / 2, size / 2],
  });
}
```

このファイルは外部 import なし（Leaflet は引数で受け取る。`L.divIcon` パターンは `src/features/map-posting/utils/cluster-icon.ts` の `createClusterIcon` と同じ）。

#### 実装パターン

- **ページ（page.tsx）**: 既存パターンにサーバーサイドフェッチを追加。`src/app/map/poster/[district]/page.tsx` がサーバーで stats をフェッチしてクライアントコンポーネントに渡すパターンと同じ
- **クライアント統合コンポーネント**: `useState` + `useCallback` で集計データを管理。登録成功後に loader を呼び出して再フェッチ
- **フック**: 既存フックにオプショナルなコールバック引数を追加するだけ。破壊的変更なし
- **地図コンポーネント**: `cityStats` が変わるたびに `useEffect` でマーカーを再描画。`src/features/map-posting/utils/cluster-icon.ts` の `createClusterIcon` と同じ `L.divIcon` パターン
- **マーカーユーティリティ**: `src/features/map-poster/utils/marker-icons.ts` の `createMarkerIconHtml` と `src/features/map-posting/utils/cluster-icon.ts` の `createClusterIcon` を参考に、`L.divIcon` で丸い集計マーカーを作成

#### 注意事項

- **既存ロジック非破壊**: `poster-placement-map.tsx` の既存の地図初期化・ピンマーカー更新ロジックは変更しない。`cityStatsMarkersRef` と新しい `useEffect` を追加するだけ
- **usePosterPlacementMap の後方互換**: `options` は optional なので、引数なしで呼び出す既存コードがあっても壊れない（現在は `poster-placement-page-client.tsx` からのみ呼ばれるが念のため）
- **zIndexOffset**: 集計マーカーの `zIndexOffset: -1000` でピン配置マーカー（デフォルト zIndex）の下に表示する。ユーザーが配置したピンが集計マーカーに隠れないようにする
- **クリックイベント**: Leaflet マーカーはデフォルトでクリックイベントをキャプチャするため、集計マーカーをクリックしても `map.on("click")` は発火しない（= ピン配置されない）。これは意図した動作
- **`avg_lat` / `avg_lng` の型**: Supabase 型再生成後、ビューの `AVG` 関数の結果は `number | null` になる可能性がある。`Number()` で変換し、`null` チェックでスキップする
- **`total_count` の型**: ビューの `SUM` 関数の結果も `number | null` になる可能性がある。`Number()` で変換する
- **Leaflet の `L` 型**: `createCityStatsMarkerIcon` では `L` を `any` 型で受け取る（dynamic import のため正確な型が使えない）。`src/features/map-posting/utils/cluster-icon.ts` の `createClusterIcon` と同じパターン
- **マーカーのツールチップ**: `direction: "top"` でマーカーの上に表示、`offset: [0, -20]` でマーカーと重ならないように調整
